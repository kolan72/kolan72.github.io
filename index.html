TMFExplorer

TMFExplorer (сокр. от TooManyFilesExplorer) - программа для работы с директориями, в которых слишком много файлов.

Когда количество файлов в директории становится слишком большим, в нее, как правило, невозможно перейти, или программа, которая используется для файловой навигации, не реагирует на действия пользователя, "зависает".

Основное преимущество программы TMFExplorer - возможность гибкой настройки фильтров при выборке файлов из таких директорий, что приводит к умеренному потреблению ресурсов операционной системы и отсутствию "зависаний", а также автоматизация таких операций, как копирование, создание Zip-архива и удаление файлов.

Системные требования

Операционные системы: Windows XP, Windows 7.

Дополнительно: Microsoft .Net Framework 4.

Описание программы

Основное назначение программы TMFExplorer - работа с директориями, содержащими слишком большое количество файлов. Путь к такой директории нужно задавать на левой панели в поле "Исходная директория". На левой же панели расположены вкладки фильтров по файлам при обновлении, обработке и удалении файлов из исходной директории. Ниже представлено подробное описание операций обновления и обработки. Полный функционал программы представлен ниже в разделе "Функциональность"

Обновление исходной директории

При открытии программы файлы из исходной директории не отображаются. Для обновления исходной директории необходимо нажать кнопку "Обновить", предварительно настроив фильтры, если в этом есть необходимость.

Время обновления может оказаться существенным и определяется доступностью ресурса исходной директории, количеством файлов, заданными фильтрами, и пр. Количество попыток обновления задается в Настройках программы. В процессе обновления его можно отменить, либо, через некоторое время после начала обновления, поставить на паузу.

По окончании обновления раскрывающаяся панель "Показать файлы" станет доступной и отобразит найденные файлы и некоторую дополнительную информацию по ним.

Обработка исходной директории

Под обработкой понимается последовательность из одной или более операций с файлами исходной директории, выполняемых друг за другом. Операции выбираются в левой панели в группе "Последовательность обработки". В случае выбора более одной операции относительный порядок их выполнения опреляется взаимным расположением - сначала выполняется операция, расположенная выше.

Состояние выполнения операций отображается в раскрывающейся панели "Лог обработки".

Фильтры применяются только к операции копирования в целевую директорию.

Операция "Создать Zip-архив файлов целевой директории" применяется ко всем файлам, находящимся на момент начала данной операции в целевой директории, безотносительно к другим возможным операциям текущей обработки. Операции копирования в целевую дирректорию и удаление в исходной по целевой допускают задание нескольких попыток. При выполнении операции удаления в исходной по целевой директории из первой удаляются только файлы, совпадающие по названию с файлами во второй.

Функциональность

  • Возможность обновления и обработки исходной директории со слишком большим количеством файлов при умеренном потреблении ресурсов операционной системы
  • Возможность настройки фильтров при обновлении, обработке и удалении из исходной директории со слишком большим количеством файлов
  • Возможность удаления файлов из исходной директории со слишком большим количеством файлов - только выделенных или всех по фильтру
  • Создание Zip-архивов по списку директорий
  • Гибкая настройка имени Zip-файла при обработке исходной директории, создании Zip-архива целевой директории или архивов по списку директорий
  • Одновременная работа с разными возможностями основного функционала программы без блокирования пользовательского интерфейса
  • Интерактивное отображение состояния выполняемых операций, лога обработки исходной директории
  • Сохраняемые настройки пользовательского интерфейса, возможность сохранения выбранных директорий для повторного обращения
  • Возможность навигации по директориям и удаления файлов для целевой директории
  • Проверка корректности пути к выбранной директории
  • Просмотр файлов целевой директории в самой программе
  • Возможность определения свободного места на ресурсе исходной директории
